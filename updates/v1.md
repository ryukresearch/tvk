# updatev1 — Location-aware search + Quick Answer card

**Status:** Planned, not built. Saved 2026-04-17.

## Goal
Let a user type a city / district / constituency into a single search bar on the Pulse tab and get a "Quick Answer" card that surfaces:
- Which constituency it is + seat #
- 2021 winner + margin
- 2026 TVK + DMK + AIADMK + NTK candidates (from existing `C26`)
- **Top 5 promises relevant to that region** (location-based manifesto highlighting)
- Jump-to-Map + Jump-to-Manifesto actions

## Why
The dashboard currently assumes users know which seat they care about. A location search is the lowest-friction entry point — "I live in Coimbatore, what's in it for me?" — and directly answers it in one screen.

## Placement
Top of the Pulse tab, above the "How TVK Benefits You" hero. Location-first, then personal.

## Data layer (draft, verified logic)

### `DIST_PERSONAS` — district → dominant persona groups
Judgment-tagged from sector concentration. `woman` added universally at render. Example slice:

```
Tiruvallur, Chennai, Chengalpattu, Tiruppur → msme/business/youth (urban industrial)
Kanchipuram, Karur, Erode, Coimbatore       → weaver heavy
Thanjavur, Tiruvarur, Nagapattinam, Theni,
Villupuram, Dharmapuri                      → farmer core
Nagapattinam, Ramanathapuram, Thoothukudi,
Kanniyakumari                               → fisher + farmer (coastal)
Salem, Madurai, Trichy, Virudhunagar        → msme/business urban mixed
```

Full 38-district map was already drafted — will be recreated when building.

### `CITY_ALIASES` — friendly names → district index
Covers the common ones: Chennai, Coimbatore/Kovai, Madurai, Trichy/Tiruchirappalli, Salem, Tirunelveli/Nellai, Tiruppur, Vellore, Erode, Thanjavur, Thoothukudi, Kanyakumari/Nagercoil, Dindigul, Karur, Hosur, Ooty, Kanchipuram, Nagapattinam, Ramanathapuram.

### Search fn — `searchLocation(q)`
Union-search across cities, districts, constituencies. Ranked: city > district > seat. Returns top 12, debounced 120ms.

### `promisesForDistrict(di, limit=5)`
Returns promises whose personas overlap with the district's dominant group. Ranked by `PROMISE_VAL.impact` descending. `woman` always included. Falls back to all-persona promises if fewer than `limit` specific matches.

## Open questions
- Should the Quick Answer card be the FIRST thing on Pulse, or live below the benefit-hero? Currently planned: above.
- Do we want the search bar to also live in the header (global)? V1 keeps it Pulse-only; header search is a v1.1 addition if used.
- `DIST_PERSONAS` tags are judgment-based, not census-derived. A future pass could replace with real census SC/ST/occupation splits.

## Not in scope for v1

- Map zoom-to-district on selection (already possible via seat click)
